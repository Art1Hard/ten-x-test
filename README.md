# Тестовое задание — Каталог курсов (10X)

Проект представляет собой **адаптивный карточный каталог с фильтрацией и поиском**, выполненный полностью на нативном JavaScript и SCSS по макету из Figma.

---

## Демонстрация

- [Ссылка на деплой](https://art1hard.github.io/ten-x-test/)
---

## Функционал

- **Pixel-perfect** верстка по макету Figma.  
- **Адаптивная «резиновая» сетка** от 1920px до 320px без жёстких брейкпоинтов.  
- **Поиск по названию карточек** в реальном времени (без перезагрузки страницы).  
- **Фильтрация по категориям** с активным состоянием выбранного таба.  
- **Динамическая генерация карточек и табов** через JS.  
- Оптимизация изображений через WebP, где возможно.  
- Структура кода организована по **BEM** и модульной архитектуре JS и SCSS.
- Автоматический расчёт количества курсов в каждой категории (в том числе и в **All**).

---

## Стек технологий

- HTML5  
- SCSS (BEM)  
- JavaScript (ES6, модули, vanilla)  

---

## Особенности реализации

- Используются **модульные JS-классы** (`TabBuilder`, `CourseBuilder`) для генерации табов и карточек.  
- Hover эффекты отключены на мобильных устройствах через медиазапросы.  
- Поиск и фильтры работают вместе: при вводе текста и выборе категории отображаются только релевантные карточки.  
- Все данные (курсы и категории) вынесены в массивы JS для простоты расширения.

---

## Что можно добавить / улучшить

- Анимации появления карточек при фильтрации.  
- Карусель карточек на маленьких экранах.  
- Lazy-loading изображений для оптимизации производительности.
- Реализацию "load more" кнопки для подгрузки курсов.
- Генерацию уникальной ссылки для каждой категории.
- Связь с бэкенд сервером для подгрузки данных с него, вместо использования моковых данных.
- Если данные приходят с бэка, то используем функцию **debounce** на поиск (меньше нагрузка на сервер при поиске).
